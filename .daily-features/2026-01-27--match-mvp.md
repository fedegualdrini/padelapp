# Feature: Match MVP (UI + WhatsApp !load)

**Status:** IMPLEMENTED

**Implemented in:** 2d328a2

**Decision:** MVP selection uses roster index only (`!load mvp <1-4>`). No name matching.

## What shipped
- DB migration adds `matches.mvp_player_id` (nullable)
- UI:
  - match edit page allows selecting MVP among the 4 match players
  - match detail page shows MVP badge when set
- WhatsApp bot:
  - `!load mvp <1-4>` sets MVP by slot index in the load session
  - MVP is stored in the match when `!load score ...` persists the match
